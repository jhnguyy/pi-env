# Code conventions

This directory holds repo-specific implementation conventions. It is intentionally progressive-disclosure: start here, then open the language or area-specific page that matches the work.

## Index

- [TypeScript](typescript.md) — exported domain values, enums, module boundaries, and type-shape preferences.
- [Effect guardrails](effect-guardrails.md) — Effect boundary, failure, resource, and composition rules.
- [Extensions](extensions.md) — extension runtime shape, lifecycle manifest, tool output, and cross-bundle singleton conventions.
- [Testing](testing.md) — capability-first hardening, independent test design, catching tests, and verification portfolios.

## How to use this directory

Before changing code in an area, check whether a matching convention page exists. Prefer these conventions over generic ecosystem defaults when they conflict.
