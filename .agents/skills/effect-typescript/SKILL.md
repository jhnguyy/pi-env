# Effect TypeScript

Before choosing an API, inspect the project's dependency, configuration, conventions, and nearby code. Use relevant `effect-solutions` guides when available, but verify against installed types. Consult source only after confirming version compatibility. Do not substitute memory for retrieval.

Use Effect where typed failures, IO, interruption, resource safety, concurrency, or dependency substitution help. Keep deterministic transforms plain TypeScript and preserve public adapters unless intentionally changing them.

Run focused tests and the project typecheck. Do not suppress new Effect language-service diagnostics unless the project explicitly baselines them.
